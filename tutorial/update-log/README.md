# 更新日志目录说明

这个目录专门记录每一次结构调整、功能新增和验证结果，目的是在未来排查问题时，能快速定位：

1. 哪次改动动了哪些文件
2. 当时为什么这么改
3. 当时做了哪些验证
4. 如果要回滚，应该优先从哪里开始

## 文件命名建议

统一使用：

```text
YYYY-MM-DD-序号-主题.md
```

例如：

```text
2026-04-21-01-structure-demo-foundation.md
2026-04-21-02-public-key-demo-article.md
2026-04-21-03-asymmetric-templates-and-support-docs.md
```

## 每条记录建议包含

1. 变更目标
2. 变更范围
3. 涉及文件
4. 验证方式
5. 潜在风险
6. 回滚起点

## 使用方式

- 发现分类、分页、图片、demo 或样式异常时，先看最近一次相关记录
- 回滚时优先按“功能块”回滚，不要一上来全仓库倒退
- 如果后续继续扩 demo、改文章组织或补自动化脚本，都建议在这里追加记录
