# 代码审计报告

这个目录用于存放本 Hexo 博客项目的周期性代码与文档审计报告。

## 命名规则

每次审计生成一个独立报告文件：

```text
YYYY-MM-DD-code-audit.md
```

示例：

```text
2026-04-25-code-audit.md
```

## 建议周期

- 文章导入工具、图片管线、发布脚本发生明显变化后，做一次轻量审计。
- 依赖升级、主题迁移、部署流程变化前，做一次完整审计。
- 报告中记录 `npm run typecheck`、`npm run build`、`npm audit` 等验证命令的结果。

## 报告内容

每份报告建议记录：

- 审计范围，以及排除的生成目录/第三方依赖目录。
- 当前工作区状态。
- 验证命令和结果。
- 按严重程度排序的问题。
- 建议后续处理动作。
