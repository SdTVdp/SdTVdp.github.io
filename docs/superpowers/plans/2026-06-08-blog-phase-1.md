# Blog Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the first technical-improvement batch for the Hexo blog: stronger classification, templates, search/SEO metadata, responsive image metadata, and CI verification.

**Architecture:** Keep Hexo + Butterfly unchanged. Add small shared TypeScript helpers for classification and post text extraction, extend the existing image pipeline without replacing the current `optimizeImageAtPath` API, and add Node verification scripts under `tools/`.

**Tech Stack:** Hexo 8, TypeScript CommonJS build, Node.js scripts, Sharp, GitHub Actions.

---

## File Structure

- `src/ts/hexo/_shared/post-classification-rules.ts`: pure category/tag rules, testable without Hexo runtime.
- `src/ts/hexo/post-classification.ts`: Hexo `before_generate` integration that applies categories and default tags.
- `src/ts/hexo/_shared/post-utils.ts`: shared post helpers, including category names and code/formula text extraction.
- `src/ts/hexo/site-enhancements.ts`: search-index and SEO description generation using shared helpers.
- `src/ts/hexo/_shared/image-pipeline.ts`: responsive image metadata generation while keeping `optimizeImageAtPath`.
- `src/ts/hexo/markdown-image-importer.ts`: image rewrite output with `srcset`, `sizes`, dimensions, and LQIP attributes.
- `src/ts/hexo/html-sanitizer.ts`: strict sanitizer support for `srcset` and `sizes`.
- `tools/test-blog-pipeline-helpers.js`: pre-build helper tests for classification and search extraction.
- `tools/verify-blog-pipeline.js`: post-build verification for generated public files.
- `scaffolds/research.md`, `scaffolds/tech-note.md`, `scaffolds/writeup.md`: article templates.
- `README.md`: scaffold usage notes.
- `.github/workflows/pages.yml`, `package.json`: CI script wiring.

### Task 1: Helper Tests

**Files:**
- Create: `tools/test-blog-pipeline-helpers.js`
- Modify: `package.json`

- [ ] **Step 1: Write failing helper tests**

Add tests that require compiled helper modules and assert path classification plus text extraction:

```js
const assert = require("node:assert/strict");
const { derivePostClassification } = require("../dist/hexo/_shared/post-classification-rules");
const { extractCodeText, extractFormulaText, getCategoryNames } = require("../dist/hexo/_shared/post-utils");

assert.deepEqual(derivePostClassification("source/_posts/archive/ctf/sample.md"), {
  categories: ["历史 WP"],
  tags: ["ctf-wp", "re-history"],
});

assert.deepEqual(derivePostClassification("source/_posts/series/re-technical-notes/vm.md"), {
  categories: ["技术笔记"],
  tags: ["re-tech-note", "vm-analysis", "crypto"],
});

assert.deepEqual(derivePostClassification("source/_posts/research/ai/transformer.md"), {
  categories: ["科研日志"],
  tags: ["ai-explainable", "deep-learning", "transformer", "self-supervised", "tda", "fluid-dynamics"],
});

assert.deepEqual(derivePostClassification("source/_posts/2025/ctf/re/old.md"), {
  categories: ["ctf", "re"],
  tags: [],
});

assert.equal(extractCodeText("<pre><code>mov eax, 1</code></pre><p>x</p>"), "mov eax, 1");
assert.equal(extractFormulaText('<span class="katex">E = mc^2</span>'), "E = mc^2");
assert.deepEqual(getCategoryNames({ categories: { data: [{ name: "ctf" }, { name: "re" }] } }), ["ctf", "re"]);
```

- [ ] **Step 2: Run test to verify it fails**

Run: `npm run compile:hexo && node tools/test-blog-pipeline-helpers.js`

Expected: FAIL because `dist/hexo/_shared/post-classification-rules.js` or exported helper functions do not exist.

- [ ] **Step 3: Add npm script**

Add `"test:blog-helpers": "npm run compile:hexo && node tools/test-blog-pipeline-helpers.js"` to `package.json`.
